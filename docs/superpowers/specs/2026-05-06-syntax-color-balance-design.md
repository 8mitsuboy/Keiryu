# Keiryu シンタックス配色調整 — デザイン仕様書

作成日: 2026-05-06

## 目的

Keiryu のシンタックス配色を、全体としてくすんだ印象に寄せつつ、`関数` を最優先で見分けやすくする。

## 背景

現状の配色は役割ごとの色責務がやや重なっている。

- `aqua` が関数と文字列の両方で強く使われる
- `purple` が型と数値系の両方で目立つ
- `orange` がキーワードとしてはやや前に出すぎる可能性がある

その結果、全体の印象はまとまっているが、関数を主役にしたいという優先順位がやや弱い。

## 方針

### 1. 全体の印象

- 高彩度には振らない
- 色相差は残すが、彩度と明度の差は抑える
- 「見分けるための色」は残しつつ、「競合する強い色」は減らす

### 2. 役割の優先順位

視線誘導の優先順位は次の通り。

1. 関数
2. 型
3. 文字列
4. キーワード
5. プロパティ / パラメータ

### 3. 色責務

- `aqua`
  - 関数名・メソッド名の主色にする
  - 文字列にも使うが、関数より静かな値に落とす
- `purple`
  - 型名・クラス名・インターフェース名・注釈系の色として残す
  - 関数より前に出ない、くすんだ紫にする
- `orange`
  - キーワード用に維持する
  - 今より少し茶寄りにして、全体の渋さに合わせる
- `green`
  - プロパティ・パラメータなど補助的な識別に留める
- `fg` / `fgMuted`
  - 演算子・一般変数・記号の基準色として使う

## 各トークンの着地

### 関数

- 最重要
- `aqua` 系を維持
- 他カテゴリより一段だけ見つけやすくする
- ただし明るくしすぎず、低彩度の青緑に留める

### 型

- 独立した役割色を残す
- `purple` 系を維持
- 関数の次に目に入るが、競合しない程度まで沈める

### 文字列

- 役割色は残す
- ただし関数と同系統で競合しないよう、より灰寄り・静かな値にする

### キーワード

- オレンジ系を維持
- 黄みを少し抑えて茶寄りにする
- 制御フローの輪郭は残すが、過剰に前に出さない

### プロパティ / パラメータ

- 補助色としての `green` を維持
- 主張は抑え、構造把握の支援に留める

## 実装方針

- 既存パレット名はなるべく維持する
- 必要なら `aquaSoft` や `purpleMuted` のような派生色を追加する
- `tokenColors` と `semanticTokenColors` で役割が逆転しないよう整合を取る
- 変更はシンタックス配色中心に留め、workbench UI 色は今回の対象外とする

## テスト観点

- TypeScript / JavaScript で関数が最も追いやすいか
- 型と関数が混在する行で競合しないか
- 文字列が関数と見分けられるか
- キーワードが茶寄りでも可読性を失わないか
- semantic token 有効時と無効時で印象が大きく崩れないか
